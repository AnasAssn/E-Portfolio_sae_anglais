Résumé du Projet (English)

The SAE2.03 project involved developing a full-stack web application to manage student absences and justification requests. The backend was built with Django and connected to a MariaDB database hosted in a virtual machine. We also configured Nginx as a reverse proxy and Gunicorn as the WSGI HTTP server to ensure a secure and performant deployment environment. This project strengthened my skills in backend development, system deployment, and teamwork.

Détails Techniques

Le backend de l'application repose sur Django, avec des fonctionnalités CRUD complètes pour les absences et justifications. La base de données MariaDB est hébergée dans une machine virtuelle (VM) Debian, configurée en réseau avec Django via des variables d’environnement sécurisées. Le déploiement de l'application s'est fait avec Gunicorn comme serveur d'application Python (WSGI), et Nginx comme reverse proxy assurant la gestion des connexions HTTP/HTTPS, la redirection vers Gunicorn, et la sécurité. Ce setup permet une architecture similaire à celle utilisée en production réelle.

Impressions et Retours

Ce que j'ai le plus apprécié, c'est l'utilisation concrète de Django et la pertinence du projet, qui ont rendu l'apprentissage plus motivant. Les difficultés rencontrées concernaient principalement la configuration du serveur, la communication entre la base de données dans la VM et Django, ainsi que le déploiement avec Nginx et Gunicorn. Malgré cela, ce projet m’a permis de progresser dans la résolution de problèmes complexes et de mieux comprendre l’architecture d’une application web déployée.
